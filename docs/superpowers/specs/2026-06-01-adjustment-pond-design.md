# 防災調整池 設計報告書作成アプリ — 設計仕様書

Date: 2026-06-01

## 概要

設計事務所・コンサルタントが防災調整池の設計計算を行い、行政提出用の報告書（PDF）を出力するWebアプリ。ブラウザ完結のSPA（サーバーレス）として実装する。

初期リリースは茨城県基準に対応し、将来的に他都道府県へ拡張できる構造とする。

---

## ターゲットユーザー

- 設計事務所・コンサルタントの設計担当者
- 利用形態：個人または少人数チームでの案件設計

## アーキテクチャ

```
src/
├── standards/           # 都道府県別の基準値・計算パラメータ
│   ├── index.ts         # 基準インターフェース定義
│   └── ibaraki.ts       # 茨城県基準（降雨強度式係数、流出係数テーブル等）
├── calc/                # 計算エンジン（純粋関数、UIに依存しない）
│   ├── hydrology.ts     # 水文計算
│   ├── structure.ts     # 構造設計計算
│   └── discharge.ts     # 放流設備計算
├── components/
│   ├── forms/           # 入力フォームコンポーネント群
│   ├── drawings/        # SVG図面コンポーネント（断面図・平面図）
│   └── report/          # 報告書レイアウトコンポーネント
├── pdf/                 # React-PDFによるPDF生成定義
├── types.ts             # 共通型定義（入力値・計算結果）
└── App.tsx              # メイン画面・タブ管理
```

**データフロー：**
`フォーム入力（useState）` → `計算エンジン（純粋関数）` → `結果・図面リアルタイム更新` → `PDF出力`

計算エンジンはUIから完全分離された純粋関数として実装し、単体テスト可能にする。

## 計算機能

### ① 水文計算（hydrology.ts）

- **降雨強度式**（茨城県基準）
  - `r = a / (t + b)^n`（Talbot型またはSherman型、地域ブロックごとに係数が異なる）
  - 確率年：5年、10年、30年、50年、100年から選択
- **流出量計算**（合理式）
  - `Q = (1/360) × C × r × A`
  - 流出係数Cは土地利用種別テーブルから選択
- **必要貯留量の算定**
  - 調整前後の流量ハイドログラフを比較
  - 三角形法による貯留量算定
  - 計画放流量（許容放流量）との差分から必要容量を決定

### ② 構造設計（structure.ts）

- 池形状の決定（矩形・台形断面）
- 法面勾配（1:n）の設定と安定確認
- 余裕高の設定（茨城県基準値を初期値として提示）
- 護岸種別（コンクリート張り、石張り等）と必要厚の計算
- 実容量 ≥ 必要貯留量 の確認

### ③ 放流設備（discharge.ts）

- **オリフィス設計**
  - `Q = C × A × √(2gH)`（C=0.6、水頭Hは設計水位から算定）
  - 孔径・個数の最適化
- **越流堰設計**
  - `Q = C × L × H^(3/2)`
  - 越流長Lの算定
- 計画放流量 ≤ 許容放流量 の確認

### ④ 図面自動生成（SVG）

- 平面図：池の外形、放流設備位置、主要寸法
- 横断面図：法面勾配、水位、余裕高、護岸構造
- 縦断面図：池底勾配、放流管位置
- 寸法線・勾配注記を自動付与

## 画面構成（5タブ）

| タブ | 主な入力項目 | 主な出力 |
|---|---|---|
| ① 基本条件 | 案件名、都道府県、流域面積、土地利用区分、計画確率年 | — |
| ② 水文計算 | 降雨継続時間、到達時間、流出係数補正 | 降雨強度・流出量・必要貯留量 |
| ③ 構造設計 | 池形状、法面勾配、余裕高、護岸種別 | 池容量・断面図プレビュー |
| ④ 放流設備 | 設備種別、孔径・個数 or 越流長 | 放流量・NG/OK判定 |
| ⑤ 報告書 | （出力のみ） | 全計算書プレビュー＋PDF出力 |

**UX方針：**
- 入力値変更でリアルタイム再計算・図面更新
- 各判定（容量不足、放流量超過、安全率不足）は該当箇所に赤字で即時表示
- タブヘッダーに ✓/✗ バッジでバリデーション状態を表示
- 未入力・エラーがある状態でもPDFプレビューは表示（エラー箇所は赤字）

---

## PDF出力仕様

- 茨城県提出様式に準拠したレイアウト
- 出力内容：表紙、設計条件一覧、各計算書（水文・構造・放流）、図面（平面・断面）
- 1ファイルにまとめて出力（React-PDFでブラウザ内生成、サーバー不要）
- A4縦を基本、図面ページは必要に応じてA4横

## 都道府県拡張設計

`standards/index.ts` に基準インターフェースを定義し、都道府県ごとに実装を差し替える構造とする。

```typescript
interface PrefectureStandard {
  name: string;
  rainfallIntensityFormula: (t: number, returnPeriod: number) => number;
  runoffCoefficients: Record<LandUseType, number>;
  freeboard: number; // 余裕高（m）
  // ...
}
```

新しい都道府県は `standards/<prefecture>.ts` を追加するだけで対応可能。
